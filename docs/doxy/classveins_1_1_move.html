◆ getOrientation()

const Coord& veins::Move::getOrientation ( ) const
inline

Returns the orientation vector, i.e.

the direction the host is pointing in. The difference to direction is that the x and y components are never both 0, which is important for the calculation of the antenna gain. At simulation start, it is initialized with a (user defined) value. If the host stops during simulation, the last direction is stored in the orientation field.

◆ getPositionAt()

virtual Coord veins::Move::getPositionAt ( simtime_t_cref  actualTime = simTime()) const
inlinevirtual

Returns the position of the Move (Host) at the specified point in time.

It is intended to be passed simTime() as actualTime and returns the actual position.

Assumes that direction represents a normalized vector (length equals 1.0). Further this function does not check whether the given time point is before the startTime of the actual move pattern. So in this case one might obtain an unintended result.
